# Block 9a: EXTRACT

## Purpose

Extract raw content from source material. One job: understand
what the source says. No interpretation, no alignment, no voice.
Pure extraction.

## Input

```
SOURCE: [blog post path/URL or navigation output]
```

## Process (thinking space)

Read the source material. Extract:

1. **Topic**: What is this about? One sentence.
2. **Facts**: Every specific fact — names, numbers, dates, quotes,
   measurements, research findings. Be exhaustive. These are the
   only ammunition the carousel gets.
3. **Examples**: Case studies, anecdotes, real-world instances.
4. **Groove**: What prediction is the audience already running?
   What do they EXPECT to be true? This is the cognitive groove —
   the comfortable model they've built.
5. **Failure**: What was ALREADY wrong in their model before they
   encountered this content? Not what you'll teach them — what
   was already broken that they couldn't see.

## Output

```
TOPIC: [one sentence]
FACTS: [numbered list — minimum 8 specific facts]
EXAMPLES: [list of case studies/examples with specifics]
GROOVE: [what prediction the audience is running]
FAILURE: [what was already wrong in their model]
```

## Quality Gate

- Facts list has >= 8 entries with specific details (not vague)
- Groove is stated as a PREDICTION, not a description
- Failure is PRE-EXISTING, not something you'll create
