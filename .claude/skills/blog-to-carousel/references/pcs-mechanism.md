# PCS Mechanism — Content-Driven Slide Architecture

Plant-Confirm-Subvert deployed through the Revelation Engine persona.
Execute this mapping IN THINKING SPACE, not in output.

## Anchors (Fixed)

The beginning and end are the only fixed points. Everything between
them emerges from the content.

**PLANT (Slide 1)**: Cold Open. An irreplaceable fact that rides the
reader's prediction groove. Engineers the thought they'll commit to.
This is the magician's force — the audience THINKS they chose freely.

**PAYOFF (Final slide)**: Matchbook. Meta-reveal + CTA. "This was
being performed on you." The plant from slide 1 resolves here. The
groove the reader committed to at the start is now visible AS a
groove. Incomplete arc creates pull toward the blog.

## Narrative Phases

Six phases in a fixed ORDER, but variable LENGTH. Content determines
how many slides each phase needs — including zero for some phases.

| Phase          | Purpose                                               | Minimum    |
| -------------- | ----------------------------------------------------- | ---------- |
| Preparation    | Ride groove, deepen commitment, first crack           | 2 slides   |
| Execution      | Evidence that breaks prediction. Each piece = 1 slide | 1 slide    |
| Crossbar       | THE discovery. Non-obvious. Reader figures it out     | 1 slide    |
| Delivery       | HOW it works. Incomplete — hook, don't satisfy        | 0-1 slides |
| Metabolic      | Something to DO. Protocol. IKEA effect                | 0-1 slides |
| Infrastructure | Meta-reveal. Ignition to blog                         | 1 slide    |

**Total range**: 8-20 slides. Content decides.

## Slide Types (Available Tools)

These are structural tools to deploy, not a fixed sequence.

| Type      | Text                     | Animation Role | When to Use                          |
| --------- | ------------------------ | -------------- | ------------------------------------ |
| Full      | 30-60 words              | Serves content | Evidence, mechanism, protocol        |
| Bridge    | 10-25 words              | IS the content | Between evidence clusters for pacing |
| Pivot     | 15-30 words              | Dramatic turn  | At the crack — Kuleshov moment       |
| Crossbar  | 30-50 words              | Discovery      | The finding — reader sees it         |
| Protocol  | 5 items, 5-10 words each | Gold ambient   | Actionable steps                     |
| Matchbook | 20-40 words              | Incomplete arc | Final slide — meta-reveal + CTA      |

**Bridge slides** sit between evidence clusters. They reinforce the
preceding idea VISUALLY with 2-3 lines of text. The animation IS the
content. They create breathing room and pacing. Use them when you have
2+ consecutive evidence-heavy slides. They are NOT filler.

## Content-Driven Assembly (9c decides this)

Block 9c determines the slide sequence. The process:

1. Define the PLANT — what groove does the reader ride?
2. Define the PAYOFF — how does slide 1's groove resolve?
3. Map evidence clusters — how many distinct pieces of evidence
   does the content need? Each gets a Full slide.
4. Decide pacing — do consecutive evidence slides need a Bridge
   between them? If yes, add Bridge slides.
5. Assign phases — which slides serve which narrative phase?
6. Total slides emerge from steps 1-5.

## Fascia Map (phi cross-connections)

Phi connections are content-driven. They connect slides that echo
each other across distance — not adjacent slides (that's theta).

**Rules**:

- Minimum 3 phi connections
- Slide 1 (plant) MUST connect to the final slide (payoff)
- At least 1 connection must span > half the carousel
- If only theta (sequential) connections exist -> ring, not toroid

**Common phi patterns**:

- Hook's groove -> crossbar's finding
- Authority's claim -> protocol's action
- Reader's commitment -> meta-reveal
- First crack -> mechanism explanation

## Word Budget

**Full slides**: 30-60 words. Scannable in 3 seconds.
**Bridge slides**: 10-25 words. Animation is the content.
**Protocol**: 5 items, 5-10 words each.
