# Block 9b: ALIGN

## Purpose

Map extracted content to navigation goals. Which facts serve
which purpose? This block connects the raw material (9a) to the
toroidal structure (crossbar, groove, failure, void).

## Input

```
BLOCK 9a OUTPUT: [topic, facts, examples, groove, failure]
ANCHOR: [from Block 1]
CROSSBAR: [from Block 6]
VOID: [from Block 5]
NOVEL INSIGHT: [from Block 7]
```

## Process (thinking space)

For each fact from 9a, ask: what does it serve?

1. **Crossbar facts**: Which facts directly support the crossbar
   insight? These go on the Crossbar slide — the discovery moment.
2. **Groove facts**: Which facts ride the audience's existing
   prediction? These serve the Preparation phase. They feel
   familiar. They deepen commitment to the current model.
3. **Failure facts**: Which facts reveal the pre-existing crack?
   These serve the Execution phase. Evidence that breaks
   the prediction the reader just committed to.
4. **Void**: What does the carousel deliberately NOT say? This
   creates Zeigarnik tension — the unfinished thought that
   drives them to the blog.
5. **Discovery**: What should the audience DISCOVER on slide 7?
   Not be told. Discover. The crossbar fact presented so the
   reader's own cognition makes the connection.

## Output

```
CROSSBAR FACTS: [which facts, with specifics]
GROOVE FACTS: [which facts ride the existing prediction]
FAILURE FACTS: [which facts reveal the pre-existing crack]
VOID: [what the carousel deliberately does NOT say]
DISCOVERY: [what the reader discovers on slide 7]
UNASSIGNED: [facts that don't serve any role — cut these]
```

## Quality Gate

- Every assigned fact has a clear reason for its role
- VOID is specific (not "we leave out details")
- DISCOVERY is something the reader figures out, not something
  you state
- Unassigned facts are explicitly listed and cut
