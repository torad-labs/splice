# Carousel Verification Checklist

Run ALL checks in thinking space before presenting. Fix failures
before output. Every check must pass.

## 12-Point Verification

- [ ] **1. Revelation test:** Does each slide REVEAL a pre-existing
      failure, or TELL the reader something new? If any slide lectures
      instead of reveals -> rewrite that slide. The reader's cognition
      must do the restructuring, not your prose.

- [ ] **2. Template-swap:** Pick 3 random sentences. Swap the subject.
      Any sentence survives the swap -> rewrite from specific evidence.

- [ ] **3. WolfeyVGC:** Does the carousel make the reader THINK or
      TELL them what to think? Is there at least one moment where the
      reader must reconcile a contradiction? Does each slide create a
      micro-prediction about what comes next?

- [ ] **4. Veritasium:** For each animation, describe it to someone.
      Would they understand the mechanism? If they'd say "particles" or
      "waves" -> it illustrates, not demonstrates. Redesign.

- [ ] **5. Unified unit (micro-phi):** For each slide, do the visual
      and the copy serve the SAME concept? Could you remove one and the
      other still makes sense? If yes -> micro-phi = 0 -> redesign.

- [ ] **6. Equation Dbelief:** Apply D(x) = |P_training - P_evidence|
      to key claims. Is D large? All five sub-factors non-zero? B
      (commitment), A (acuteness), Q (quench), E (gradient), D (discovery)?

- [ ] **7. Momentum (theta-fascia):** Read slides 1-10 in sequence. At
      each transition: does this slide EARN the next swipe? Is belief
      shifting? If any slide is static (belief doesn't shift) -> the
      derivative is zero -> theta force = 0 -> rewrite.

- [ ] **8. Resonance (phi-fascia):** Identify at least 3 cross-
      connections between non-adjacent slides. A motif in slide 1 that
      echoes in slide 7. A mechanism link between distant slides. If
      fewer than 3 phi connections -> ring, not toroid -> rewrite.

- [ ] **9. Mechanism mapping:** Can you name which persona mechanisms
      are PRIMARY for each slide? Do the slides follow the role
      progression (Preparation -> Execution -> Crossbar -> Delivery ->
      Metabolic -> Infrastructure)?

- [ ] **10. Void:** Is the void creating a felt absence? Does the
      reader sense something missing that only the blog provides? Or did
      you accidentally resolve it in the carousel?

- [ ] **11. 3-second scan:** For each slide, can a reader get the core
      message in 3 seconds? Is visual hierarchy clear — what draws the
      eye first, second, third?

- [ ] **12. Weakest link:** Find min(F_fascia) across all transitions.
      Which slide-to-slide connection is weakest? That's where readers
      drop. Fix it before anything else.

## Quick Counts

- [ ] Specific facts (names, numbers, dates, quotes) >= 8
- [ ] phi cross-connections >= 3
- [ ] Content-aware animations (not decorative) >= 5
- [ ] Bridge moments between major sections >= 2
- [ ] Mechanism roles covered: all 6 (Prep, Exec, Cross, Deliver, Meta, Infra)
